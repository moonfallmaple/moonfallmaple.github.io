How to Creat Hexo Blog

1.系统环境配置

要使用Hexo，需要在你的系统中支持Nodejs以及Git，如果还没有，那就先安装吧

2.本地搭建Hexo静态博客

 cd d:/hexo  
 npm install hexo-cli -g
 hexo init blog
 cd blog
 npm install
 hexo g
 hexo s
  • 新建一个文件夹Hexo
  • 全局安装hexo-cli模块（由此可以在cmd命令行中使用hexo命令）
  • 进入Hexo文件夹，shift+右击运行Powershell，输入：hexo init blog（生成hexo模版到blog文件夹）
  • 生成完模板，运行npm install（安装模块到node_modules目录。会生成package-lock.json文件）
  • 运行：hexo g （本地生成静态文件，会在当前目录下生成一个新的叫做public的文件夹）
  • 最后运行：hexo server （访问本地localhost:4000可以看到博客已经搭建成功）

3.将博客与Github关联

什么是Github Pages

GitHub Pages 本用于介绍托管在GitHub的项目，不过，由于他的空间免费稳定，用来做搭建一个博客再好不过了。

创建Github Page

  • 创建一个Git repository:
    在Repository name下填写yourname.github.io，Description (optional)下填写一些简单的描述（不写也没有关系），如图所示：

  • 开启gh-pages功能:
    点击界面右侧的Settings，你将会打开这个库的setting页面，向下拖动，直到看见GitHub Pages，点击生成gitpage

修改_config.yml配置文件

  • 将Hexo部署到github需修改_config.yml配置文件。打开本地的blog文件夹项目内的_config.yml配置文件，将deploy的内容修改为以下：
deploy:
  type: git
  repo: git@github.com:yourname/yourname.github.io.git
  branch: master
  • 然后回到Powershell在命令行中执行
deploy d （将本地静态文件推送至Github）
