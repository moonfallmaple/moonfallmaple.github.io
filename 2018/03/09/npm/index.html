npm

简介

npm(全称Node Package manager,即“node包管理器”)是一个命令行下的软件，用来管理和安装node模块。
npm不需要单独安装。在安装Node的时候，会连带一起安装npm。但是，Node附带的npm可能不是最新版本，最好用下面的命令，更新 到最新版本。

  • @latest 表示最新版本
  • -g 表示全局安装
  • 命令的主干是npm install npm，也就是使用npm安装自己。之所以可以这样，是因为npm本身与Node的其他模块没有区别。

npm常用命令

npm init（新建项目）

  • 通过命令行创建一个目录，假定叫做hello-world，并通过cd进入该目录。
  • 通过npm init命令来创建一个 package.json 文件。该文件是工程开始的第一步，它记录了项目所需要的各种模块以及项目的配置信息（比如名称、版本、许可证等元数据）。

npm init：运行命令它会向用户提问一系列问题，如果你觉得不用修改默认配置，一路回车就可以了。在回答一堆问题后输入 yes 就会生成以下所示内容的package.json 文件。

npm install （安装模块）

Node模块采用npm install命令安装。

每个模块可以“全局安装”，也可以“本地安装”。

  • 全局安装: 将模块安装到系统目录中，各个项目都可以调用。一般来说，全局安装只适用于工具模块，比如eslint和gulp。

  • 本地安装: 将模块下载到当前项目的node_modules子目录，然后只有这个项目才能通过require()方法调用这个模块。

指定依赖

package.json里面有两个对象是用来指定依赖的：

  • “dependencies”:列出 生产 环境下的依赖
  • “devDependencies”:列出 开发 环境的依赖

在npm install命令后设置–save和save-dev可以省掉手改修改package.json文件的步骤。
